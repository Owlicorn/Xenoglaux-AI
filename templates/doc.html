API Documentation

Explore the Xenoglaux AI API with detailed endpoints, parameters, and examples. Integrate the model into your applications.

API Endpoints

POST /api/chat

Chat with Xenoglaux AI using our conversational endpoint.

Parameters
message string (required) - User message to send to Xenoglaux AI
max_length integer (optional) - Maximum tokens to generate (default: 512)
temperature float (optional) - Generation temperature (0.1 to 2.0)
POST /api/generate

Direct text generation without conversational context.

Parameters
prompt string (required) - Input prompt for text generation
max_length integer (optional) - Maximum tokens to generate
temperature float (optional) - Generation temperature
POST /api/clear

Clear the current conversation history and context.

No parameters required
GET /api/info

Get detailed system information and model status.

No parameters required
GET /api/health

Health check endpoint to verify API availability.

No parameters required
GET /api/stats

Get training data statistics and model metrics.

No parameters required

API Features

Auto-scaling model architecture
Free generation (1-512+ tokens)
Context-aware responses
Dual data sources (lang_model.txt + MongoDB)
Conversation context awareness
Thinking process visualization

Example Usage

Here's an example of how to use the chat endpoint with JavaScript:

fetch('/api/chat', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    message: 'Tell me a story about a robot discovering emotions',
    max_length: 256,
    temperature: 0.8
  })
})
.then(response => response.json())
.then(data => {
    console.log('AI Response:', data.response);
  // Handle the response
})
.catch(error => {
  console.error('Error:', error);
});

The API will return a JSON response with the AI's answer and thinking process.
