## How to Contribute

### 1. Reporting Issues
- Check existing issues before opening a new one.
- Provide a clear title and detailed description.
- Include steps to reproduce the issue if applicable.

### 2. Submitting Pull Requests (PRs)
- Fork the repository and create a branch for your feature/bugfix.
- Write clear, concise commit messages.
- Make sure your code follows the existing code style.
- Run tests (if any) before submitting a PR.
- Reference the related issue in your PR description, e.g., `Closes #issue_number`.

### 3. Code Style
- Follow the coding conventions used in the project.
- Keep functions and modules small and focused.
- Add comments where necessary to explain tricky parts.

### 4. Branching & Workflow
- Use `main` (or `master`) for stable code only.
- Create feature branches: `feature/your-feature` or `bugfix/your-bugfix`.
- Always rebase or merge the latest `main` before opening a PR.

### 5. Code Review
- PRs will be reviewed by maintainers.
- Be ready to discuss and adjust your code based on feedback.
- Keep discussions polite and constructive.

### 6. Communication
- Use GitHub issues or discussions for questions.
- Avoid off-platform communication for project decisions.

### 7. License
- Make sure your contributions comply with the project license.
